Lab 12.1 Work: PHP Session Management

Applications that use PHP sessions to maintain state across multiple pages or AJAX requests. These exercises demonstrate how to store data on the server, track user progress, and destroy sessions when the game ends.

  • Three‑Page Guessing Game

    A classic guessing game spread over three pages. On the first page, the user sets a numeric range. The second page generates a secret number (stored in $_SESSION) and asks for a guess. The third page reveals whether the guess was correct and destroys the session. Session management ensures the secret number is never exposed to the client.

  • Extended Guessing Game – Try Again

    An enhancement of the three‑page game where the session persists until the user guesses correctly. If the guess is wrong, a “Try Again” link returns to the guess page with the same secret number. Only a correct guess destroys the session, allowing multiple attempts without reloading the range.

  • Slot Machine with AJAX and Session

    A fully interactive slot machine that uses AJAX for spins and PHP sessions to track credits. The user starts with 10 credits, can bet any amount (subject to limits), and wins multipliers for two or three matching symbols. The session stores the current credits and is destroyed when the player runs out of money – the game ends automatically. Includes loading feedback and button constraints during AJAX requests.

Lab 11.1 Work: AJAX and JSON Data Fetching

Asynchronous web applications that fetch data from the server without reloading the page. These exercises demonstrate AJAX requests, JSON responses, and dynamic UI updates with loading feedback and constraints.

  • Password Strength Checker (AJAX)

    Real‑time password validation using AJAX. As the user types, a PHP script checks length, uppercase, lowercase, digits, and symbols. The input border turns green/red instantly, and a loading spinner appears during the request. The input remains enabled while typing (debounced) and previous requests are cancelled.

  • City Population Search (JSON)

    One‑page app that queries the world database for cities within a population range. The form sends min/max values via AJAX to a PHP endpoint that returns a JSON array of matching cities. While waiting, the search button is disabled and a loading spinner is shown. Results are displayed below the form in a clean, formatted list.

Lab 10.1 Work: Advanced PHP and Database Integration

Further exploration of PHP, including custom functions, array manipulation, and database access using PDO. These exercises build on the server-side foundations from Lab 9.1.

    A collection of reusable PHP functions for common tasks such as input sanitization, validation, and mathematical operations. This exercise emphasizes modular code design and function reusability.

  • Exercise 2: Associative Arrays and Iteration

Lab 9.1 Work: Server-Side Scripting with PHP

Introduction to server-side programming using PHP. These exercises demonstrate form handling, validation, and dynamic content generation.

  • Slot Machine

    A simple slot machine simulation that randomly displays three fruit images. If two match, you win a small prize; all three match means jackpot. The page includes a "PUSH BUTTON" link to spin again.

  • Tip Calculator

    A one-page app that calculates a restaurant tip. It validates server name, email (with confirmation), bill amount, tip percentage, and credit card number (16 digits). Errors are displayed clearly, and the form retains entered values.

  • Multiplication Table Generator

    A configurable multiplication table. The form lets you set the table title, number of rows/columns, operation (x, +, -), and colors. Missing parameters default to 10x10 with multiplication. The generated table is styled and includes a warning when defaults are used.

  • Configurable Slot Machine

    An enhanced slot machine where you can set the number of wheels (1-10) and a bet amount. If all wheels match, the payout is bet x 100 x number of wheels; if at least half match, the payout is bet x number of wheels. The form submits to itself and displays the result along with the spun fruit images.

Lab 8.2 Work: Animation and Team Logo

Collaborative exercises exploring canvas animations and creating an animated team logo. These projects were developed in partnership with Alex Chen.

  • Canvas Animation Warm-up

    A modified version of the basic canvas animation. The ball now moves down and to the right, shrinks gradually, and transitions from black to red as it moves. This exercise served as a warm-up to get comfortable with modifying existing animations and working collaboratively.

    • Ball moves down and right (positive x and y speeds).
    • Radius decreases over time for a shrinking effect.
    • Red color component increases from 0 to 255, shifting from black to red.
    • Maintains collision detection to stop at canvas edges.
    • Demonstrates incremental changes to an existing codebase.
  • Animated Team Logo - PixelForge

    A custom animated splash screen for our web development team, PixelForge. The animation features two independently moving objects (a square tracing a Lissajous curve and a circle in a circular orbit) over a subtle pixel-grid background, combined with the team name and tagline. Created collaboratively with Alex Chen.

    • Two moving objects (square and circle) animated simultaneously using requestAnimationFrame.
    • Each object is defined as a separate JavaScript class with update and draw methods.
    • Static background grid reinforces the tech theme.
    • Team name and tagline displayed in monospace font with glow effects.
    • Color palette uses blues, purples, and white, matching the desired modern, clean aesthetic.

Lab 7.1 Work: Object-Oriented Programming and Client-side Storage

A collection of exercises exploring JavaScript classes, objects, and persistent storage using localStorage. These projects demonstrate encapsulation, JSON serialization, and state management across browser sessions.

  • Number Guessing Game with OOP

    A classic number guessing game implemented using a custom GuessingGame class. The game selects a random number and tracks the number of guesses, providing feedback on whether each guess is too high, too low, or correct. The game ends after 10 incorrect guesses and automatically resets upon a correct guess.

    • Custom GuessingGame class with constructor and guess method.
    • Tracks guess count and last guess result.
    • Visual feedback with color-coded messages.
    • Automatic reset after correct guess or 10 incorrect attempts.
    • Clean, professional interface.
  • Persistent Guessing Game with localStorage

    Enhancement of the guessing game that saves the game state to localStorage after every change. When the user returns to the page, the game resumes exactly where they left off, including the secret number, guess count, and last guess.

    • Game state (secret number, guess count, last guess) saved as JSON in localStorage.
    • Automatic restoration of previous game on page load.
    • Game state cleared upon correct guess or game over to ensure a fresh start.
    • Seamless integration with the existing OOP design.
    • Demonstrates client-side persistence using localStorage.
  • 500 Balls Animation with Bounce and Gravity

    A canvas animation featuring 500 randomly generated balls with individual colors, speeds, and positions. The Ball class has been extended with bounce and gravity methods, creating realistic bouncing arcs within the canvas boundaries.

    • 500 Ball objects created with random positions, speeds, and colors.
    • Bounce method handles wall collisions, reversing speed components.
    • Gravity method applies a constant downward acceleration.
    • Continuous animation with requestAnimationFrame/setInterval.
    • Demonstrates object-oriented design and canvas animation techniques.
  • Mobile Memo Pad with 50 Pages

    A mobile-friendly memo application that stores up to 50 pages of text in localStorage. Users can navigate between pages, save changes, revert to saved content, and the app remembers the last viewed page across sessions.

    • Textarea for entering memos, with Save and Revert buttons.
    • 50 pages stored as a JSON array in localStorage.
    • Previous/Next buttons to flip through pages.
    • Buttons are disabled when no changes are pending.
    • Responsive design with a professional mobile aesthetic.
  • Interactive DOM Manipulation with Mouseover Deletion

    An advanced DOM manipulation example featuring buttons to add various elements (divs, link fragments, checkerboards) and a mouseover listener that deletes any div when hovered over. Demonstrates dynamic element creation and event handling.

    • Button to add numbered divs that can be removed by mouseover.
    • Button to insert a fragment containing a div with a link.
    • Button to generate a 2x2 checkerboard using CSS Grid.
    • Mouseover listener attached to all divs (existing and new) for deletion.
    • Showcases DOM manipulation, event delegation, and CSS styling.

Lab 6.1 Work: Canvas Interaction and Drawing

A series of canvas-based exercises exploring mouse and keyboard control, persistent drawing, and a complete paint application.

  • Street View on a Starry Night

    A simple line-drawing tool that lets the user build a street scene step by step. Features include:

    • House drawing (Button 2): Each click adds a new house to the right of the previous one, using a persistent global variable to track the next x-coordinate.
    • Car drawing (Button 3): Places a simple car under the most recently drawn house.
    • Random star placement (Button 4): Draws a star at a random position in the “sky” (top part of the canvas) with a random bright colour each time the button is clicked.
    • Clean, monochrome line style for houses and cars, while stars are filled with vibrant random hues.
    • Boundary checking prevents houses from being drawn beyond the canvas edge.

    This lab exercise demonstrates canvas API usage, function modularisation, and state management with global variables.

  • Pong with Mouse-Controlled Paddle

    A basic Pong simulation where the left paddle follows the mouse vertically. Features include:

    • Canvas with black background, two paddles, a dashed net, a ball, and a score display at the top.
    • Left paddle Y position updates in real-time using mousemove events.
    • Right paddle remains static, demonstrating a simple two-player mock-up.
    • All drawing is done inside the load event and redrawn on every mouse move.
    • Highlights canvas drawing fundamentals and mouse-event handling.

    This exercise lays the groundwork for more advanced game loops and collision detection.

  • Keyboard-Controlled Circle

    An extension of the street scene canvas where a green circle can be moved with the arrow keys. Features include:

    • A green circle (radius 15) starts at the centre and moves 10px per arrow key press.
    • All previously drawn houses, cars, and stars are stored in arrays and redrawn after each movement, preserving the scene.
    • Arrow key events are captured and the circle's position is clamped to stay within the canvas.
    • Integrates with the existing button-based drawing functions (houses, cars, stars) without interference.
    • Demonstrates persistent state management and keyboard event handling alongside canvas drawing.

    This exercise shows how to combine multiple interaction modes in a single canvas application.

  • Interactive Paint Application

    A fully-featured paint app with multiple brush types, color selection, and modifier keys, organised into separate HTML, CSS, and JavaScript files. Features include:

    • Four brush shapes/sizes: large/small circle and large/small square, drawn centred at the mouse.
    • Color picker: an <input type="color"> lets the user choose any drawing colour; the current colour is displayed in the status bar.
    • Controlled drawing: painting only occurs while the mouse button is held down (using mousedown/mouseup flags).
    • Eraser modifier: hold Ctrl while dragging to erase a square area (via clearRect).
    • Size doubling: hold Alt while drawing to temporarily double the current brush size.
    • Clear canvas: pressing Backspace or Delete (with confirmation) or the “clear canvas” button erases everything.
    • Modular code: HTML structure, CSS styling, and JavaScript logic are split into three files (paintApp.html, paintStyles.css, paintJS.js) for maintainability.

    This exercise brings together event handling, canvas manipulation, and a polished user interface - a strong foundation for the upcoming assignment.
